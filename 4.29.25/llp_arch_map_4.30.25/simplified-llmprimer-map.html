Component Details

Bookmarklet (bkm_llmPrimer_v5.0_4.17.25.js)

The entry point script that executes when the user activates the bookmarklet:

  • Checks if panel already exists in the DOM
  • Toggles panel visibility if it exists
  • Loads the KB loader script if panel doesn't exist
  • Sets up panel initialization once loader is ready
  • Handles basic UI functions like theme toggling

Knowledge Base Loader (kbLoader_llmPrimer_v5.3_4.17.25.js)

Manages the loading and organization of knowledge base content:

  • Defines knowledge base categories (designReview, research)
  • Defines knowledge base entries with metadata
  • Creates the global llmPrimerRegister function as a JSONP callback
  • Loads KB files via script tags to bypass CORS restrictions
  • Validates configurations and tracks loading status
  • Initializes the UI panel once content is loaded
  • Handles loading errors and timeouts

Knowledge Base Files (kb_*.js)

Individual content files containing prompts and templates:

  • Simple structure with a single primary function call
  • Each file calls llmPrimerRegister with its ID and markdown content
  • Example files: kb_darkDeceptivePatterns.js, kb_usabilityHeuristics.js
  • Content is stored as markdown strings for easy formatting
  • Files are loaded via JSONP to avoid cross-origin restrictions

UI Components

The interface elements displayed to the user:

  • Category selector: Shows available knowledge base categories
  • KB selector: Shows knowledge bases within the selected category
  • Action buttons: "Inject Prompt" and "Copy Prompt"
  • Theme toggle: Switches between light and dark mode
  • Panel controls: Minimize/maximize and close buttons

Data Flow Summary

  1. Activation: User clicks bookmarklet, which either toggles existing panel or initiates loading process
  2. Configuration: KB loader defines available categories and knowledge bases
  3. Content Loading: Individual KB files are loaded and register their content
  4. Panel Creation: UI panel is created with dropdowns and buttons
  5. User Selection: User selects category → knowledge base → action
  6. Content Delivery: Selected KB content is injected into target text area or copied to clipboard
