LLM Primer Bookmarklet Flow Diagram

The UX Bookmarklet is a JavaScript tool designed to streamline the creation of pre-prompts for UX and accessibility considerations. Here's a breakdown of how it works:

1. Bookmarklet Activation

The user clicks the bookmarklet in their browser's bookmark bar, triggering the injection of the UI panel into the current webpage.

Key Processes:

  • Self-executing function initialization
  • Variable setup for knowledge base URLs
  • Style definitions for UI components
  • Dynamic panel creation and injection

2. Knowledge Base Loading

The system loads external knowledge base files based on user selection (Accessibility, UX, or Research).

Key Processes:

  • AJAX or fetch requests to load JavaScript files
  • Fallback loading mechanisms for compatibility
  • Cache checking for previously loaded data
  • Error handling for failed loads
  • UI updates to reflect loading status

3. Feature Selection

The user selects a specific feature from the loaded knowledge base to focus the pre-prompt on.

Key Processes:

  • Dynamic population of feature dropdown
  • Feature metadata retrieval from knowledge base
  • UI updates based on selection
  • Event handling for selection changes

4. Pre-prompt Generation

The system generates a structured pre-prompt based on the selected feature and knowledge base data.

Key Processes:

  • Template selection based on knowledge base type
  • Extraction of feature-specific data (considerations, principles)
  • Random selection of relevant personas
  • Markdown formatting for structured output
  • Domain-specific section construction

5. Pre-prompt Delivery

The generated pre-prompt is delivered via direct injection into the page or copied to the clipboard for manual pasting.

Key Processes:

  • DOM targeting to find input fields
  • Direct text injection with event simulation
  • Clipboard API integration for copy functionality
  • Visual feedback for successful operations
  • Panel cleanup and removal

Knowledge Base Structure

The external knowledge bases are structured as JavaScript objects with well-defined properties:

Features Collection:

  • Feature definitions
  • Design considerations
  • Principles & best practices
  • WCAG guidelines (accessibility)
  • User scenarios & stories

Persona Categories:

  • Category mappings by feature
  • Persona groupings
  • Domain-specific classifications
  • Accessibility needs taxonomy
  • User role definitions

Template Generation:

  • Prompt generation functions
  • Markdown formatting logic
  • Section assembly process
  • Domain-specific request templates
  • Persona selection algorithms

UI Panel Components

The injected UI panel includes:

  • Knowledge base selector dropdown
  • Feature selector dropdown
  • "Inject Pre-prompt" button
  • "Copy Pre-prompt" button
  • Status indicator
  • Theme and style toggles
  • Close button

This system allows users to quickly generate structured, domain-specific pre-prompts that improve AI responses related to UX, accessibility, and research best practices.
