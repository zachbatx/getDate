LLM Primer & Knowledge Base: Real-World Implementation Guide

How It Works: End-to-End Workflow

The LLM Primer & Knowledge Base system transforms general-purpose LLMs into specialized domain experts through a simple bookmarklet workflow:

  1. Bookmarklet Activation:
  • User visits an LLM interface (like ChatGPT or Claude)
  • Clicks the bookmarklet in their browser's bookmark bar
  • A floating panel appears overlaid on the current webpage
  1. Knowledge Selection:
  • User selects a knowledge domain (e.g., "Dark Patterns", "Usability Heuristics", "Accessibility")
  • Chooses a specific feature or framework within that domain
  1. Prompt Injection:
  • User clicks "Inject Prompt" to insert specialized knowledge into the LLM session
  • The system formats and injects domain expertise, evaluation frameworks, and structured guidance
  • The LLM now has the context of a domain expert in the selected area
  1. Enhanced Interaction:
  • User can now interact with the LLM as if speaking with a domain specialist
  • The LLM applies the injected knowledge to provide expert-level guidance

Real-World Application Examples

Example 1: Dark Pattern Evaluation

Workflow:

  1. User activates bookmarklet while in an LLM session
  2. Selects "Dark Patterns" from the knowledge base dropdown
  3. Clicks "Inject Prompt"
  4. The LLM is now primed with comprehensive dark patterns taxonomy and evaluation framework

Sample User Interactions:

USER: I've uploaded screenshots of our subscription cancellation flow. Can you analyze it for potential dark patterns?

LLM: I'll analyze your subscription cancellation flow for dark patterns. Please describe the steps in detail or share the screenshots.

USER: [Uploads screenshots of a multi-step cancellation process]

LLM: [Conducts structured analysis following the dark patterns framework]

PATTERN NAME: Roach Motel

CATEGORY: Obstruction

LOCATION: Cancellation flow, steps 3-5

ATTRIBUTES: Asymmetry, Restriction

USER IMPACT: Forces users through multiple unnecessary confirmation screens, creating friction specifically for the cancellation process while sign-up is simple.

LEGAL CONSIDERATIONS: May violate consumer protection regulations regarding unfair business practices.

RECOMMENDATIONS: Simplify the cancellation process to match the simplicity of the sign-up process. Limit to one confirmation screen.

USER: Can you evaluate this form design for accessibility issues?

[Conducts structured accessibility evaluation]

ISSUE: Color contrast ratio

WCAG REFERENCE: 1.4.3 Contrast (Minimum)

SEVERITY: Critical

FINDING: The light gray text on light blue background has a contrast ratio of approximately 2.5:1, below the 4.5:1 minimum required.

IMPACT: Users with low vision or color vision deficiencies will struggle to read form labels.

RECOMMENDATION: Increase the contrast ratio by using darker text or a lighter background.

User Guidance: Getting the Most from the System

Best Practices for Users

  1. Clear Context Provision:
  • Upload screenshots or provide detailed descriptions of interfaces
  • Specify the complete user flow or journey when relevant
  • Indicate the target audience or user group when applicable
  1. Specific Requests:
  • Ask for evaluations of specific aspects (e.g., "Focus on the checkout flow")
  • Request analysis from particular perspectives (e.g., "Evaluate as a financial regulator")
  • Specify output format preferences (e.g., "Provide a severity-ranked list")
  1. Iterative Refinement:
  • Ask for detailed analysis of identified issues
  • Request specific recommendations for improvements
  • Upload revised designs for re-evaluation

Follow-up Strategies

Once you receive an initial evaluation, you can:

  1. Request Prioritization:
  • "Which issues should we address first based on user impact?"
  • "Can you rank the identified problems by implementation effort vs. impact?"
  1. Ask for Alternatives:
  • "What are some alternative designs that would resolve these issues?"
  • "Can you suggest specific improvements for the most critical problems?"
  1. Seek Implementation Guidance:
  • "How could we implement your recommendation for [specific issue]?"
  • "What would an accessible version of this component look like?"
  1. Request Comparison:
  • "How does this design compare to industry best practices?"
  • "Can you evaluate our implementation against relevant regulations?"

Real-World Benefits

This system democratizes access to specialized expertise by allowing:

  1. Solo Practitioners to perform expert-level evaluations without specialized training
  2. Design Teams to integrate ethical and accessibility considerations early in the process
  3. Researchers to apply established methodologies consistently
  4. Product Managers to ensure compliance with industry standards
  5. Startups to access expertise that might otherwise be financially out of reach

By transforming general-purpose LLMs into domain-specific assistants, the system makes specialized knowledge more accessible while ensuring evaluations follow established best practices and methodologies.
